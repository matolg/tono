## 1. Обзор

**Tono** — мобильное приложение для музыкантов на Flutter. MVP включает два инструмента:

- **Хроматический тюнер** — определяет высоту звука через микрофон, отображает ноту, октаву и отклонение в центах.
- **Метроном** — задаёт темп (40–220 BPM), размер такта, воспроизводит акцентированные и обычные удары.

Запланированы (не реализованы): тренажёр слуха, тренажёр аккордов.

**Целевая платформа:** Android (minSdk 26 / Android 8.0+). iOS — в планах.

## 4. Слои архитектуры

Каждая фича организована по трём слоям:

```
┌────────────────────────────────┐
│        Presentation            │  Widgets, Screens
│  (ConsumerWidget / Consumer-   │  ref.watch() → rebuild
│   StatefulWidget)              │  ref.read() → actions
├────────────────────────────────┤
│           Domain               │  Notifier + State
│  (AsyncNotifier / Notifier)    │  Бизнес-логика, оркестрация
├────────────────────────────────┤
│            Data                │  Сервисы, движки
│  (PitchDetector,               │  Работа с платформой,
│   MetronomeEngine, etc.)       │  аудио, хранилище
└────────────────────────────────┘
```

**core/** содержит инфраструктуру, не привязанную к конкретной фиче: провайдеры глобального состояния, реестр инструментов, утилиты.

**shared/** содержит переиспользуемые UI-компоненты без бизнес-логики.

## 5. Фичи

### 5.1 Тюнер (`features/tuner/`)

| Компонент | Ответственность |
|-----------|----------------|
| `TunerNotifier` | Управляет состоянием прослушивания, обрабатывает частоты, хранит опорную высоту A4 (432–444 Гц) |
| `TunerState` | Статус (idle / listening / detected / error), нота, октава, частота, отклонение в центах |
| `PitchDetector` | Захват аудио (PCM16, 44100 Гц) через `record`, определение высоты через `pitch_detector_dart` |
| `TunerScreen` | Отображение стрелки, ноты, октавных точек, селектора опорной частоты |

Алгоритм определения высоты: YIN / MPM (pitch_detector_dart). Входящие кадры фильтруются по уровню энергии во избежание ложных срабатываний на тишину.

При попадании в диапазон ±5 центов — тактильная обратная связь (haptic feedback).

### 5.2 Метроном (`features/metronome/`)

| Компонент | Ответственность |
|-----------|----------------|
| `MetronomeNotifier` | BPM (40–220), количество долей (2–8), единица доли, тап-темпо, персистентность |
| `MetronomeState` | Статус (playing / stopped), текущая доля |
| `MetronomeEngine` | Изолят с дрейфово-скорректированным таймером, воспроизведение через SoLoud |
| `MetronomeScreen` | BPM-слайдер, кнопки ±1, индикаторы долей (они же кнопка тап-темпо) |

Синхронизация времени: используется дрейфово-скорректированный алгоритм (не `Timer.periodic`) для предотвращения накопления погрешности. Живое изменение BPM передаётся в изолят без перезапуска.

### 5.3 Главный экран (`features/home/`)

Отображает карточки инструментов из `ToolRegistry`. Недоступные инструменты помечаются бейджем «Скоро».

### 5.4 Настройки (`features/settings/`)

Переключение языка (ru / en) и темы (светлая / тёмная / системная). Настройки сохраняются через SharedPreferences и восстанавливаются при запуске.

## 6. Управление состоянием

Весь state management реализован через **Riverpod 3.x** с кодогенерацией (`riverpod_generator`).

```
┌────────────────────────────────────────┐
│          Глобальные провайдеры         │
│  localeProvider    (LocaleNotifier)    │
│  themeModeProvider (ThemeModeNotifier) │
├────────────────────────────────────────┤
│          Провайдеры фич                │
│  tunerProvider     (TunerNotifier)     │
│  metronomeProvider (MetronomeNotifier) │
└────────────────────────────────────────┘
```

- `ref.watch(provider)` — подписка на изменения, вызывает перестройку виджета.
- `ref.read(provider.notifier).method()` — однократный вызов метода.
- Провайдеры глобального состояния (локаль, тема) инициализируются при старте в `main.dart`.

## 7. Потоки данных

### Тюнер

```
Микрофон
  └─> record (PCM16, 44100 Гц)
        └─> фильтр по энергии
              └─> pitch_detector_dart (YIN/MPM)
                    └─> MusicUtils.frequencyToNote()
                          └─> TunerNotifier → TunerState
                                └─> TunerScreen (ref.watch)
```

### Метроном

```
Пользователь (start)
  └─> MetronomeNotifier.start()
        └─> Isolate (drift-corrected timer)
              └─> 'beat' message → main thread
                    └─> MetronomeNotifier._fireBeat()
                          ├─> SoLoud.play(accent | tick)
                          └─> MetronomeState (currentBeat++)
                                └─> MetronomeScreen (ref.watch)
```

### Локализация и тема

```
App (ConsumerWidget)
  ├─ ref.watch(localeProvider)    → MaterialApp.locale
  └─ ref.watch(themeModeProvider) → MaterialApp.themeMode
```

## 9. Хранение данных

Все данные — локальные, облачного хранилища нет.

| Данные | Хранилище |
|--------|-----------|
| Язык интерфейса | SharedPreferences |
| Тема (light/dark/system) | SharedPreferences |
| BPM метронома | SharedPreferences |
| Количество долей и единица доли | SharedPreferences |

Ключи и чтение/запись инкапсулированы внутри соответствующих Notifier-классов.

## 12. Ключевые архитектурные решения

| Решение | Обоснование |
|---------|-------------|
| **Feature-first структура** | Каждый инструмент самодостаточен: domain, data, presentation — в одном модуле |
| **Riverpod** | Компилируемая безопасность типов, удобная инъекция зависимостей, нет boilerplate Provider |
| **Изолят для метронома** | Главный поток свободен от таймера; дрейфово-скорректированный алгоритм исключает накопление погрешности |
| **Разделение аудио-библиотек** | `record` — для захвата (тюнер), `flutter_soloud` — для воспроизведения (метроном); каждая оптимизирована под свою задачу |
| **ToolRegistry** | Централизованный реестр позволяет добавлять новые инструменты без изменения главного экрана |
| **Material 3 + собственная палитра** | Тёплые земляные тона (C4856A / D4957A), поддержка светлой и тёмной темы |
| **Полная локализация** | Все строки в ARB-файлах, нет захардкоженного текста; поддерживаются ru и en |
