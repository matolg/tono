## Команды разработки

```bash
# Установить зависимости
flutter pub get

# Запустить на подключённом устройстве / эмуляторе
flutter run

# Статический анализ (должен быть чистым; ошибки — блокер)
flutter analyze --fatal-infos

# Тесты
flutter test --reporter expanded

# Кодогенерация (после изменений в файлах с @riverpod-аннотациями)
dart run build_runner build --delete-conflicting-outputs

# Сборка release AAB для Android
flutter build appbundle --release
```

Перед каждым коммитом убедись, что `flutter analyze` и `flutter test` проходят без ошибок.
Не пушь в `master` с непроходящим CI.

## Соглашения по коду

### Структура фичи

Каждая фича организована по трём слоям:

```
presentation/   — виджеты и экраны
domain/         — Notifier + State, бизнес-логика (без UI-зависимостей)
data/           — сервисы и движки (платформа, аудио, хранилище)
```

Presentation не обращается к data напрямую — только через domain.

### Riverpod

- Каждый Notifier — в своём файле `*_notifier.dart`.
- После изменений в файлах с `@riverpod`-аннотациями — запускай кодогенерацию.
- `ref.watch()` — для подписки (виджеты), `ref.read(provider.notifier).method()` — для действий.

### Локализация

Все строки интерфейса — только через `AppLocalizations`. Хардкод в UI недопустим.
При добавлении строк обновляй оба файла (`app_ru.arb` и `app_en.arb`), затем `flutter gen-l10n`.

### Навигация

GoRouter. Маршруты объявлены в `lib/app/router.dart`. Не используй `Navigator.push` напрямую.

### Новые инструменты

Регистрируй в `lib/core/tool_registry/tool_registry.dart`.
Недоступные добавляй с `isAvailable: false` — карточка появится с бейджем «Скоро».
